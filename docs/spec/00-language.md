# 0. 言語の定義

## 0.1 Seseragi とは何か

Seseragi は、式中心・静的型付き・デフォルト不変の関数型言語です。

言語の中心は、軽い構文そのものではなく、次の意味論にあります。

1. 関数はカリー化され、部分適用できる。
2. 型の違う状態は代数的データ型で表す。
3. 欠如、失敗、非同期、副作用を値と型に現す。
4. パターンマッチは網羅的である。
5. 型クラスにより、データ表現とは独立に振る舞いを抽象化する。
6. 外部言語の都合は境界で変換し、Seseragi 内部の意味を変えない。

代表的な関数宣言は次の形です。

```seseragi
fn add x: Int -> y: Int -> Int =
  x + y
```

## 0.2 設計原則

### 式中心

`if`、`match`、ブロックは式であり、値を返します。関数本体も式です。
制御構造のためだけの文と、値を作る式を分けません。

### デフォルト不変

`let` は再代入できない名前を導入します。record、struct、ADT、Array、List の値も
生成後に変更できません。状態変化は新しい値、または状態を明示するライブラリ型で
表現します。

### 暗黙に危険を隠さない

通常の値として `null` と `undefined` は存在しません。回復可能な欠如と失敗は
それぞれ `Maybe` と `Either` で表します。非同期の失敗は `Task<E, A>` の `E` に
現れます。

### 意味はバックエンドから独立する

整数の除算、評価順序、recordの構造、Effectの遅延性などは、生成先の言語に
よって変化しません。外部ランタイムの値は、外部連携境界を通して変換します。

### 診断できない魔法を作らない

オーバーロード、型推論、暗黙の辞書選択は、コンパイラが一意に説明できる場合だけ
成功します。候補が複数ある場合、適当に一つを選ばずコンパイルエラーにします。

### 読める密度を保つ

Seseragiの簡潔さは、記号を減らすことではなく、データの形と処理の流れを一目で読めることを
意味します。Array / Listのliteralとcomprehension、pipeline、Signalの読み書きなど、概念を
直接表す構文は言語のsurface identityとして扱います。

独自構文には、型、評価順序、展開先、失敗条件を必ず定めます。短さのために副作用、部分関数、
暗黙変換を隠してはなりません。通常関数と同じ程度に明瞭でない糖衣構文は追加しません。

## 0.3 プログラム

ソースファイル一つが一つのモジュールです。モジュールは import と宣言だけを
トップレベルに持ちます。任意のトップレベル式は許可されません。

実行可能プログラムは次の名前を公開しなければなりません。

```seseragi
pub effect fn main -> Unit
with AppEnv
fails AppError =
  program
```

これは `main: Unit -> Effect<AppEnv, AppError, Unit>` を宣言する糖衣です。`AppEnv` はclosedな
structural service recordです。`main` の実行方法、environment provider、終了コードへの写像は
host target規約に従いますが、Effect自体の意味は言語仕様に従います。

## 0.4 この仕様に存在しないもの

次は Seseragi の中核機能ではありません。

- 変数への再代入と可変フィールド
- `null`、`undefined`、暗黙の nullable 型
- 暗黙の数値変換、truthy / falsy
- 非網羅的なパターンマッチ
- 例外を通常のエラー処理に使う仕組み
- 任意の union 型と intersection 型
- `perform` / `handle` を持つalgebraic effect system
- user-definedな汎用構文マクロと手続きマクロ
- ユーザー定義のprefix / postfix演算子
- `return` による関数途中からの脱出

これらに似た外部値や外部挙動は、外部連携境界で明示的に扱います。

`deriving` とtemplate Stringは、仕様で入力と展開意味を閉じた限定的なコンパイル時機能です。
任意の構文木を受け取ってcodeを生成する汎用マクロ機構ではありません。
